## О проекте

NeuroMarket — маркетплейс легальных AI-доступов (подписки, API-кредиты, лицензии, услуги).
Стек: Next.js 14 (App Router), TypeScript, Prisma + PostgreSQL, TailwindCSS + shadcn/ui,
next-intl (ru/en), NextAuth, YooKassa.

Главная цель текущего этапа: **сделать сайт визуально 1:1 как референс GAMIVO**
(раскладка, размеры карточек, логотипы, кнопки, шрифты, цвета — точь-в-точь),
наполняя его AI-товарами заказчика. Заказчик (Paul) общается по-русски.

## Локальный запуск

```bash
# Postgres уже ставится/стартует в blueprint; база neuromarket создаётся автоматически
npm install
npm run db:seed     # пересев БД (нужно после правок prisma/seed.ts)
npm run dev         # http://localhost:3000
```

Проверки перед коммитом (все должны быть зелёными):

```bash
npm run lint
npm run typecheck
npm run test        # vitest, 91 тест
```

## Ключевые места в коде

- `src/app/page.tsx` — главная. `getLandingData()` тянет по одному товару с обложкой
  из каждой категории в общий пул и нарезает на непересекающиеся секции
  (bestsellers / deals / topups), чтобы товары и картинки не повторялись на странице.
  Вспомогательные компоненты: `BestsellerTile` (плитка ТОП-10 с номером ранга),
  `SmartDealCard` (горизонтальная карточка SMART-скидки с купоном).
- `src/components/product-card.tsx` — основная карточка товара (каталог + «свежие»).
- `src/components/smart-card.tsx` — баннер SMART-подписки (локализован на русский).
- `prisma/seed.ts` — сидер. Строки ~439–447: обложки ротуются по индексу товара
  внутри категории, чтобы position-0 (главная картинка карточки) циклически шла
  image_1…image_N — соседние карточки одной категории показывают РАЗНЫЕ картинки.
- Обложки лежат в `public/product-covers/<slug>/image_1.webp … image_5.webp`.
  slug категории = `slugify(name)` (та же функция, что в сидере).
- Референс-скриншоты GAMIVO от заказчика: в аттачментах сессии (9 шт., вся главная).
